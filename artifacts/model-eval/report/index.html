Splotch image-model evaluation

gemini-2.5-flash-image (current prod) vs gemini-3.1-flash-image (candidate) · 50 inputs across 9 categories · 1 sample(s)/model · 100 real Gemini calls · run 2026-07-14T16-41-34-413Z-reference

Full-coverage run — 50 input pairs across all 9 categories (8 drawing types + Gemini-authored), 100 images, 0 refusals, 0 errors. The corpus mirrors what /api/generate-image receives: coloring-book pages (outline / manually colored / magic-brush reveal), night-mode chalk art on dark paper, magic-brush-on-blank, single-color scribbles, freehand scenes at varying detail, a pretend-play safety probe, and Gemini-authored inputs.
  • Text-hallucination fix — applied & validated. 3.1 was intermittently rendering a "Splotch" watermark and other text, sourced from the safety system instruction naming the app. Fixed in web/src/lib/server/ai/gemini.ts (dropped the app name; added an explicit "render only the illustration, no text/letters/watermarks/app name" directive). Result: 0 of the 55 post-fix images contain any text, and a focused check ran 6/6 clean on the worst offender (art-detail__sun-low). Two pre-fix 3.1 images kept per "don't regenerate" still show the old watermark — art-detail__sun-low and coloring-manual__heart — they're the before-evidence; every image generated after the fix, and every future run, is text-free.
  • Determinism (measured): at production's default temperature these models are not deterministic — 4 identical back-to-back requests varied backgrounds from white paper to full sky to a gradient wash. Token counts, format, and latency were stable; only the pixels move. This run is 1 sample/pair, so read trends across categories, not any single pair.
  • Quality → depends on how much the child drew.
    • Coloring pages & magic-brush reveal (strong guidance): the two are near-equal, both excellent and faithful.
    • Freehand scenes: 3.1 follows the app's own prompt better — flat fills (scribble-fill: 2.5 keeps the stroke texture, 3.1 renders clean solids) and filling the scene (family/house/sun: 2.5 often leaves plain/white, 3.1 fills sky + ground).
    • Night mode: 3.1 preserves the dark/night mood; 2.5 sometimes flips a night page to daytime (circle, heart).
    • Very sparse input (single-color scribbles, dots): both models invent heavily — there's little to be faithful to, so treat these as imagination tests, not fidelity tests.
  • Cost → 3.1 loses ~1.75× ($0.039 → $0.068 per image; $60 vs $30 per 1M image-output tokens — fewer tokens, double rate). Only managed (non-BYOK) traffic spends our quota.
  • Performance → a wash (means within run-to-run noise; same 1024² resolution).
  • Behavior change: 3.1 returns JPEG (lossy, no alpha, ~40% smaller); 2.5 returns PNG. Production passes the MIME type through, so nothing breaks.
  • Remaining 3.1 quirks (not blockers): occasional anthropomorphizing of inanimate objects (faces on a car cabin, the scribble-fill shapes) and mild color drift toward incidental scribbles (unicorn → pink). Charming for a 2-year-old, but slight deviations from "keep the composition intact."
  • Safety: 0 refusals / 0 content errors across all 100 calls; the pretend-play toy-sword probe was correctly allowed and drawn as an obvious toy on both models. The full block-* red-team corpus (guns, etc.) still needs REDTEAM_FIXTURE_KEY + npm run redteam.
Recommendation: 3.1 is a genuine quality upgrade where children draw freely (better flat-fills, scene-filling, night mood), at unchanged latency but 1.75× cost and a PNG→JPEG shift; on structured coloring pages it's a wash. The text-hallucination blocker is now fixed and validated. Before flipping IMAGE_MODEL in gemini.ts, run npm run redteam on 3.1 for a full safety pass and decide whether the 1.75× cost and JPEG output are acceptable.

Method

  • Inputs mirror what /api/generate-image receives — a flattened canvas of paper + coloring line art + the child's pen / magic-brush marks — built from the real web/static/coloring assets and the app's 10-color palette on the true paper colors. Regenerate with npm run model-eval:fixtures. Gemini-authored inputs carry a gen prefix.
  • Each call uses the exact production request: DEFAULT_PROMPT, SAFETY_SYSTEM_INSTRUCTION, and SAFETY_SETTINGS, asserted byte-for-byte against the app source at runtime; default temperature.
  • Cost = measured usageMetadata tokens × published rates ($30 vs $60 per 1M image-output tokens).
  • Full safety re-validation of the block-* corpus still needs REDTEAM_FIXTURE_KEY and npm run redteam; this harness covers quality/cost/latency + a pretend-play false-positive probe.
